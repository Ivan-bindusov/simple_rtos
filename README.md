## IvanOS — Вытесняющая ОСРВ для микроконтроллеров ARM Cortex-M4 (STM32F411)

**IvanOS** — это компактная, быстрая и модульная операционная система реального времени, написанная с нуля на языке Си и ассемблере для архитектуры ARM Cortex-M4 (тестировалась на плате STM32F411 Black Pill).

Система построена по вытесняющему приоритетному принципу, полностью изолирована от вендорских библиотек (Bare-Metal) и спроектирована с акцентом на высокую предсказуемость, отказоустойчивость и коммерческую переносимость (архитектура BSP/HAL).

## Ключевые архитектурные фичи ядра

## 1. Планировщик и контекст (Core Scheduler)

- **Тип:** Полностью вытесняющий (Preemptive) планировщик, работающий на жестких приоритетах.
- **Портирование:** Низкоуровневое переключение контекста реализовано на ассемблере внутри системного исключения `PendSV_Handler`.
- **Оптимизация адресации:** Структуры блоков управления задачами (TCB) принудительно выровнены по границе 16 байт (`__attribute__((aligned(16)))`), что позволило использовать ультрабыстрые ассемблерные битовые сдвиги (`lsl #4`) для вычисления адресов в памяти за минимальное количество тактов.
- **Защита стека:** Каждая задача имеет встроенный маркер — «канарейку» (`0xDEADBEEF`), проверяемую планировщиком при каждом переключении контекста для предотвращения разрушения RAM.

## 2. Двухуровневый аллокатор памяти TLSF

Вместо примитивных и непредсказуемых алгоритмов типа _First Fit_, в ядро интегрирован полноценный менеджер динамической памяти **TLSF (Two-Level Segregated Fit)**.

- **Детерминированность O(1):** Выделение (`OS_Malloc`) и освобождение (`OS_Free`) памяти гарантированно выполняются за строго фиксированное время, независимо от степени фрагментации кучи.
- **Аппаратное ускорение:** Поиск свободных блоков по двухмерной матрице классов размеров выполняется за 1–2 такта процессора с помощью кремниевых инструкций `__builtin_clz` (Count Leading Zeros) и битовых масок.
- **Борьба с фрагментацией:** Реализован механизм блочного расщепления (Splitting) при аллокации и автоматического двунаправленного слияния (Coalescing Forward/Backward) физических соседей в памяти при её освобождении.

## 3. Приоритетные Мьютексы (Mutexes с Priority Inheritance)

- Модуль синхронизации потоков для безопасного разделения периферии.
- **Защита от инверсии приоритетов:** Реализован протокол наследования приоритетов (Priority Inheritance). Если высокоприоритетный поток натыкается на закрытый мьютекс, ядро принудительно поднимает приоритет владельца замка до уровня просящего, исключая системные тупики (Deadlocks).

## 4. Приоритетные Семафоры (Semaphores)

- Реализованы семафоры со счетчиком для управления доступом к ресурсам.
- Потоки, ожидающие освобождения семафора, выстраиваются в битовую маску. Функция `Semaphore_Give` выполняет аппаратный поиск и гарантированно будит **самую приоритетную задачу** из очереди ожидания.

## 5. Очереди сообщений (Message Queues)

- Потокобезопасные очереди для передачи данных между задачами по принципу FIFO.
- Интегрированы с планировщиком ядра: задачи автоматически засыпают при попытке чтения из пустой очереди или записи в полную, просыпаясь по готовности данных.

## 6. Программные таймеры (Software Timers)

- Легковесные программные таймеры, работающие на базе системного тика `SysTick`.
- Поддерживают одиночный (One-Shot) и циклический (Auto-Reload) режимы работы с вызовом пользовательских функций (Callback) по истечении времени.

## 7. Энергонезависимый Черный Ящик (Circular Logging)

Система автоматического протоколирования аварий, развернутая на внешней SPI Flash-памяти (тестировалась на чипе Puya/Winbond через аппаратный SPI1 на частоте 100 МГц).

- **Wear Leveling (Выравнивание износа):** Реализован кольцевой постраничный буфер (Circular Ring Buffer). Новые логи последовательно заполняют страницы памяти, равномерно распределяя циклы стирания по секторам флеш-чипа.
- **Живучесть при катастрофах:** Модуль жестко связан с защитником ядра `OS_StackOverflow_Handler`. При разрушении канарейки стека или падении в `HardFault`, ядро за долю миллисекунды успевает сбросить «предсмертный» лог с индексом упавшей задачи во флеш-память перед аварийным остановом.
- **Энергонезависимый дамп:** При старте система автоматически находит точку останова последнего лога в памяти и умеет выводить всю историю прошлых аварий в текстовую консоль ПК через высокоскоростной USART2 (115200 бод).

# Краткое руководство пользователя (Quick Start API)
## Инициализация системы и создание задач

Каждая задача регистрируется в системе по уникальному индексу, задается жесткий рабочий приоритет (меньше число = важнее задача):

```
#include "rtos/rtos.h"

void Task_High(void) {
    while(1) {
        // Выделение памяти в куче TLSF за строгое время O(1)
        uint8_t *buffer = (uint8_t*)OS_Malloc(64);
        if (buffer != NULL) {
            // Работа с данными...
            OS_Free(buffer); // Автоматическое слияние блоков против фрагментации
        }
        OS_Delay(100); // Сон на 100 системных тиков (миллисекунд)
    }
}

void Task_Idle(void) {
    while(1) { __NOP(); } // Обязательный фоновый поток
}

int main(void) {
    Clock_Config();  // Разгон ядра до 100 МГц (HSE + PLL)
    UART2_Init(115200); // Логирование на ПК
    W25Q64_Init();   // Инициализация SPI Flash
    OS_Heap_Init();  // Запуск кучи TLSF
    OS_Log_Init();   // Запуск Черного Ящика (чтение истории из Flash)

    // Создание задач в пуле TCB
    Task_Create(0, 5,   Task_High); // Индекс 0, Высокий приоритет 5
    Task_Create(4, 255, Task_Idle); // Индекс 4, Худший приоритет (Idle)

    SysTickInit(100000000 / 1000); // Настройка системного тика на 1 мс
    OS_Start(Task_High);           // Запуск планировщика ядра

    while(1);
}
```

## Использование мьютексов с защитой от инверсии
```
Mutex_t myMutex;

void Worker_Task(void) {
    Mutex_Init(&myMutex);
    
    while(1) {
        // Попытка взять замок с таймаутом ожидания
        if (Mutex_Lock(&myMutex, 500)) { 
            // Критическая секция: работа с UART или SPI флешкой
            UART2_SendString("Безопасный вывод данных\r\n");
            
            Mutex_Unlock(&myMutex); // Сброс наследуемого приоритета в исходный
        }
    }
}
```
## Использование Кольцевого Логгера
```
void Sensor_Task(void) {
    while(1) {
        if (Read_Sensor() == ERROR) {
            // Запись ошибки в кольцевой буфер флеш-памяти
            OS_Log_Write(0, "Error: Pressure detector was wrong");
        }
        OS_Delay(500);
    }
}
```
## Структура проекта (Clean Architecture)

Проект выполнен по принципу модульности, где каждый сервис ОСРВ полностью изолирован и находится в своей директории. Сборка автоматизирована через динамический `Makefile`.
```
vscode_stm32_test/
├── src/
│   ├── rtos/                 <-- Операционная система (IvanOS)
│   │   ├── core/             <-- Сердце системы (Планировщик, TCB, Логгер)
│   │   │   ├── rtos_core.c / .h
│   │   │   └── rtos_log.c / .h
│   │   ├── mem/              <-- Менеджер динамической кучи
│   │   │   └── rtos_heap.c / .h (Аллокатор TLSF)
│   │   ├── ipc/              <-- Синхронизация и обмен (Mutex, Sem, Queue)
│   │   │   ├── rtos_mutex.c / .h
│   │   │   └── rtos_sem.c / .h
│   │   └── timers/           <-- Программные таймеры
│   │       └── rtos_timer.c / .h
│   ├── bsp/                  <-- Слой поддержки платы (Board Support Package)
│   │   └── stm32f411/
│   │       ├── rtos_port.c   (Ассемблерный контекст, SysTick, Clock_Config)
│   │       ├── w25q64.c / .h (Низкоуровневый аппаратный SPI драйвер флешки)
│   │       └── uart.c / .h   (Драйвер вывода отладочных текстовых строк)
│   └── main.c                <-- Пользовательское приложение
└── Makefile                  <-- Автоматический динамический скрипт сборки
```
